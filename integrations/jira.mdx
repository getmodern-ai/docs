## What the Jira integration enables

| Capability           | Description                                                                                     |
| -------------------- | ----------------------------------------------------------------------------------------------- |
| Automation workflows | Streamline issue creation, status updates, project management tasks, and user access management |
| Ticket Syncing       | Sync tickets between Modern and Jira                                                            |

Anything defined in the [Jira REST API](https://developer.atlassian.com/cloud/jira/platform/rest/v3/intro/) can be accessed through Modern.

***

## Jira Configuration

Connecting Jira to Modern is straightforward and requires no manual configuration on the Jira side. The integration uses OAuth to securely connect your Jira instance with Modern.

### Prerequisites

* You must have admin permissions in your Jira instance
* Access to install apps in your Jira workspace

***

## Modern Configuration

1. In Modern go to **Apps → Available → Jira → Connect**
2. Click the **Connect** button to initiate the OAuth flow
3. A new window will open directing you to your Jira instance

4. Review and accept the integration permissions and scopes
5. Click **Accept** to complete the connection
6. You'll be redirected back to Modern with the integration successfully connected

That's it! No additional configuration is needed. Modern can now access your Jira instance through the authorized OAuth connection.

## Additional Notes

### Ticket Syncing

To enable ticket syncing, you need to install the Modern app in your Jira instance and configure the sync settings.

<Warning>
  You must first connect your Jira instance to Modern using the OAuth flow described in the [Modern Configuration](#modern-configuration) section above before proceeding with ticket syncing setup.
</Warning>

#### Step 2-6: Configure Sync Settings

Once the app is installed, complete the remaining configuration steps in Modern:

1. **Select Project** - Choose the Jira project to sync tickets with
2. **Select Request Type** - Choose the request type for creating tickets in Jira
3. **Priority Mapping** - Map each Jira priority to a Modern priority option
4. **Status Mapping** - Map each Jira status to a Modern status option
5. **Sync Strategy** - Configure how and when tickets should be synced
