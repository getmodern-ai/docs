## About Ramp

Ramp is a finance automation platform that combines corporate cards with expense management, bill payments, and accounting integrations. Connecting Ramp to Modern enables automated expense workflows, transaction monitoring, and financial reporting directly from your service desk.

## What the Ramp integration enables

| Capability           | Description                                                                       |
| -------------------- | --------------------------------------------------------------------------------- |
| Access Management    | Create, update, and manage users and their permissions within your Ramp account   |
| Automation workflows | Query transactions, manage expense policies, retrieve spending insights, and more |

Anything defined in the [Ramp API](https://docs.ramp.com/developer-api/v1/overview) which uses the available permission scopes can be accessed through Modern.

***

## Ramp Configuration

### Prerequisites

* You must have **Admin or Business Owner** access to your Ramp account
* For default setup: No additional configuration needed
* For advanced setup: A Ramp Developer App (instructions below)

  <Tab title="Advanced Setup (BYOC)">
    ## Bring Your Own Ramp Developer App

    Use this method if you need to use your own Ramp Developer App credentials. This uses the OAuth 2.0 Client Credentials flow (2-legged OAuth) and doesn't require callback URLs.

    <Note>
      This method is designed for custom apps. We recommend using the default setup unless you have specific requirements for your own app credentials.
    </Note>

    <Steps>
      <Step title="Create a Ramp Developer App">
        1. Navigate to **Settings** → **Developer API** in your Ramp account
        2. Click **Create New App**
        3. Name your app (e.g., "Modern Integration"), accept the Terms, and click **Create**
        4. Under **Grant types**, ensure **Client Credentials** is enabled
        5. Under **Scopes**, configure the scopes you need for your use case
        6. Copy your **Client ID** and **Client Secret**

        <Info>
          Client credentials flow doesn't require redirect URIs since authentication happens server-to-server without user interaction.
        </Info>
      </Step>

      <Step title="Connect in Modern">
        1. In Modern, go to **Apps → Available → Ramp → Connect**
        2. Click **Edit scopes** to select your desired scopes (you must select at least one optional scope)
        3. Click the three-dot menu (⋮) next to "Scopes" and select **"Use your own Ramp app"**
        4. Enter your **Client ID** and **Client Secret**
        5. Click **Connect**

        <Warning>
          The scopes you select in Modern must also be enabled in your Ramp Developer App settings.
        </Warning>
      </Step>

      <Step title="Confirm connection">
        The integration will authenticate using your credentials and connect immediately without requiring OAuth authorization.

        <Check>
          Verify the connection is successful by checking the integration status in your Modern apps list.
        </Check>
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Using the Ramp Integration

Once connected, you can use Ramp actions in your Modern workflows:

```typescript Example: List Ramp users theme={null}
import { ramp } from "modern-integrations";

const listUsers = async (ctx: ramp.RampIntegration) => {
  const response = await ramp.apiRequest({
    path: "/developer/v1/users",
    method: "GET"
  }, ctx);
  
  return response.data;
};
```

For complete API documentation, refer to the [Ramp Developer API Reference](https://docs.ramp.com/developer-api/v1/overview).

## Troubleshooting

<AccordionGroup>
  <Accordion title="'Not Authorized' or 'Business not authorized' error">
    This error occurs when:

    * You don't have Admin or Business Owner permissions in Ramp
    * The Modern app is not properly configured in Ramp

    **Solution**: Ask a Ramp Admin or Business Owner to complete the authorization, or contact [support@modern.com](mailto:support@modern.com) for assistance.
  </Accordion>

  <Accordion title="Scope permission errors">
    If you receive errors about missing permissions:

    * Verify the scope is included in your authorization
    * For BYOC mode: Ensure the scope is enabled in your Ramp Developer App settings
    * Some scopes may require special approval from Ramp

    **Solution**: Reconnect the integration with the required scopes.
  </Accordion>

  <Accordion title="Token expired errors">
    If your integration stops working after some time:

    * Your access token may have expired
    * The `offline_access` scope is now automatically included and required for all connections

    **Solution**: Reconnect the integration. The `offline_access` scope will be automatically included to enable automatic token refresh.
  </Accordion>

  <Accordion title="Invalid scope errors">
    If you see errors about invalid scopes when typing scope names:

    * You may have mistyped a scope name
    * The scope might not exist in Ramp's API

    **Solution**: Click **Edit scopes** to see a complete list of valid scopes you can select from, or refer to the [Ramp API documentation](https://docs.ramp.com/developer-api/v1/overview) for scope details.
  </Accordion>
</AccordionGroup>
