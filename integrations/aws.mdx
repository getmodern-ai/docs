## About AWS

Amazon Web Services (AWS) is a public-cloud platform, offering on-demand compute, storage, database, and networking services. Connecting AWS to Modern lets you automate everyday cloud-ops and security workflows—from listing IAM roles to spinning down unused EC2 instances—directly from chat.

## What the AWS integration enables

| Capability           | Description                                                       |
| -------------------- | ----------------------------------------------------------------- |
| Access Management    | Create, update, and manage users and their permissions            |
| Automation workflows | Manage resources, create roles, and automate lifecycle management |

***

## AWS configuration (in the AWS console)

Follow these steps to create a cross-account role that Modern can assume. The role's policies determine exactly which AWS APIs Modern can call.

1. **Open the modal in Modern**
   In the Modern UI navigate to **Apps → Connect AWS**. Leave this window open; it shows the **Modern AWS Account ID** and **External ID** you will need shortly.

2. **Create a new IAM role**\
   In AWS go to **IAM → Roles → Create role** and choose **Another AWS account** as the trusted entity.\
   Enter the **Account ID** shown in Modern, then tick **Require external ID** and paste the **External ID**.

<Tip>
  In the JSON below, replace `<replace-with-your-external-id>` (including the angle brackets `<` and `>`) with the actual **External ID** shown in the Modern **Connect AWS** modal.
</Tip>

```json  theme={null}
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "AWS": "arn:aws:iam::992382851720:root" },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": { "sts:ExternalId": "<replace-with-your-external-id>" }
      }
    },
    {
      "Effect": "Allow",
      "Principal": { "AWS": "arn:aws:iam::992382851720:root" },
      "Action": "sts:TagSession"
    }
  ]
}
```

## Modern Configuration

1. Return to the **Connect AWS** modal in Modern.
2. Enter your **AWS Account ID**, a **display name** for the account (e.g. "Production" or "Sandbox"), and the **Role ARN** of the role you just created, then click **Submit**.

<Note>
  The **Your AWS Account Name** field is a human-readable label used to identify this AWS account within Modern. It does not affect the integration — you can enter any name that helps you distinguish between accounts.
</Note>
